Tone of Voice Engine

An experimental tool for exploring and documenting tone standards

This walkthrough covers:

  • The user experience and flow
  • How the tool works behind the scenes
  • The JavaScript configuration and logic
  • How content standards are generated

The Problem

Most tone of voice frameworks are too abstract:

  • "Be warm and approachable"
  • "Sound professional but friendly"
  • "Use a conversational tone"

But what does that actually mean when you're writing?

The Gap

There's a disconnect between high-level tone descriptors and the specific linguistic devices that create those effects.

The Approach

This tool breaks tone down into linguistic devices — the actual mechanisms that create tonal effects.

Five Tone Spectrums

  • Playful ↔ Serious
  • Casual ↔ Formal
  • Direct ↔ Indirect
  • Warm ↔ Cool
  • Confident ↔ Humble

User Flow: Step 1

Select a Spectrum

Users start by choosing which tone spectrum they want to explore.

Five tabs at the top of the interface: • Playful ↔ Serious • Casual ↔ Formal • Direct ↔ Indirect • Warm ↔ Cool • Confident ↔ Humble

Each spectrum maps to different linguistic characteristics.

User Flow: Step 2

Position the Slider

A slider lets users position themselves on the spectrum from 0-100.

Position 0 = Strongly left (e.g., Playful) Position 50 = Balanced middle Position 100 = Strongly right (e.g., Serious)

As you move the slider, the interface updates in real-time to show relevant linguistic devices.

User Flow: Step 3

Explore Sub-Dimensions

Each spectrum breaks down into 4 sub-dimensions.

Example: Playful vs. Serious includes:

  • Language flow — Fluid vs. Rigid
  • Energy level — Lighthearted vs. Earnest
  • Rhythmic quality — Bouncy vs. Measured
  • Creative expression — Imaginative vs. Conventional

User Flow: Step 4

View Linguistic Devices

Each sub-dimension shows 3-5 specific devices based on your slider position.

Example devices for "Playful":

  • Contractions
  • Exclamation points
  • Informal interjections
  • Varied sentence length

Click any device to see its definition, examples, and UX impact.

User Flow: Step 5

Generate Content Standard

Once you've configured your tone positioning, click Generate content standard.

This opens a modal where you provide:

  • Scope level (Global, Product, Team, Feature)
  • Owner team
  • Domain
  • Content types

The tool then generates a complete markdown document with your tone configuration documented.

Behind the Scenes: Data Structure

The spectrumData Object

All tone configurations live in a JavaScript object:

const spectrumData = { 'playful-serious': { title: 'Playful vs. Serious', leftLabel: 'Playful', rightLabel: 'Serious', dimensions: { fluidity: { title: 'Language flow', leftLabel: 'Fluid', rightLabel: 'Rigid', devices: { left: ['Contractions', ...], right: ['Formal punctuation', ...] } }, // ... more dimensions } }, // ... more spectrums }

Behind the Scenes: Device Definitions

The deviceDefinitions Object

Each device has detailed information:

const deviceDefinitions = { 'Contractions': { definition: 'Shortened forms of word combinations that create conversational tone.', examples: [ "You'll love this feature", "Can't find what you're looking for?", "We've got you covered" ], impact: 'Makes interface copy feel more approachable and human.' }, // ... more devices }

Behind the Scenes: Slider Logic

Dynamic Device Selection

The slider value (0-100) determines which devices to show:

if (sliderValue <= 16) { // Strongly left devices = dimension.devices.left; } else if (sliderValue <= 33) { // Moderately left devices = [ ...dimension.devices.left.slice(0, 4), ...dimension.devices.right.slice(0, 1) ]; } else if (sliderValue <= 49) { // Slightly left - mix devices devices = [ ...dimension.devices.left.slice(0, 3), ...dimension.devices.right.slice(0, 2) ]; }

This creates 6 granular positions across the spectrum.

Generating Markdown Output

The tool assembles a complete content standard document:

The Output: Content Standard

What Gets Generated

A complete markdown document with:

  • Metadata — ID, version, owner, dates
  • Context — Domain, content types, scope
  • Tone definition — Clear positioning on spectrum
  • Device guidance — Specific linguistic tools to use
  • Examples — Correct and incorrect usage
  • Validation rules — JSON/YAML for automation
  • Testing criteria — Checkable requirements
